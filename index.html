Variables

Any programming you will build in your career will have data, and it will be hard to treat with data without anything that can maintain this data.

So that we use variables, with variables I can maintain my program's data and treat with users data also.

Example:

let x = 1; let y = 5; let z = y + x;

In the last example, you can see the word let this word we used with JavaScript language to tell that we well make variable.

All of (x, y, z) is our variable.

let x = 1; // variable x contain the value 1. let y = 5; // variable y contain the value 5. let z = y + x; // variable z contain the result of x + z = 1 + 5 = 6.

Print functions

When a programmer work on his project, he all the time needs to see the result of his code, to be sure if he is in the right way or not.

So in any programming language, you will have a print function that can print the result of your code, usually, we didn't use it to print anything to our users.

JavaScript print functions:

let x = "Ahmed Love FreeCodeCamp"; console.log(x);

In our code above you maybe know this part: let x = "...",It's our variable, but this part (console.log(x)) is out print function.

My console will output this value:

Ahmed Love FreeCodeCamp

Of course, the console.log() function is not the only printing function in JavaScript, and any language usually has multiple of them.

Comments

In large projects you will have many program sentences and other programmers working with you in the same project, Usually, the programmers needed something to tell them what is your code do without reading your code to avoid wasting time.

So that every language has a way to write a comment.

The common stile to write a comment in one line in several programming languages:

// your comment

The style above we used also in JavaScript, and this is a common style to write a comment:

/* your comment your comment your comment */

You don't use comments to explain your code for other programmers only, you use them for you also to organize your code.

If conditional concept

When you built your program you may need to customize your code for several cases, in this situation we used the if statement

Say that I have a move for adults only or children with their patents, and I didn't want only children to see the movie.

To solve the problem we will use the if statement because we have many cases.

let isAdult = true; let whithParents = false; if (isAdult == true) // this statement to sure if the watcher is adult or a child. { console.log("You can watch the movie"); // this what will excute if the watch is an adult. } else if (isAdult == false & whithParents = true) // this statement to sure if the child watcher with his parents or not. { console.log("You can watch the movie"); // this what will excute if the watchet is a child and with his parents. } else // this statement will be true any way if both previouse statements doesn't exist. { consoele.log("You can't watch the movie") //this what will excute if the previouse statements doesn't exist. }

Loop concept

In many situations you will repeat your code with a type of progress many times, I can say that is what we do most of our coding time.

So that we have the loop concept, Every programing language has it's own loop statement.

The common keyword for loop concept is for.

The code below is a loop for printing numbers from 1 to 10 in JavaScript.

for( let i = 1 ; i <= 10; i++ ) { console.log(i); }
